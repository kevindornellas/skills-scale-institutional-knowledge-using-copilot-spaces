# OctoAcme — Communication Plan Template

## Purpose
Provide a reusable template for defining how a project or release communicates with its stakeholders. A completed Communication Plan reduces coordination overhead, ensures the right people receive the right information at the right cadence, and establishes clear ownership for each communication channel.

> **Owner:** Project Manager, in collaboration with Product Manager.
> See [Roles & Personas](./octoacme-roles-and-personas.md) for full role descriptions.

## Communication Cadence

Define the recurring touchpoints for this project.

| Meeting / Update | Frequency | Participants | Owner | Channel / Format |
|---|---|---|---|---|
| Project kickoff | Once | All roles | Project Manager | Meeting |
| Weekly status update | Weekly | PM, PdM, Eng Manager, key stakeholders | Project Manager | Email / doc |
| Team standup | Daily (or per team agreement) | Delivery team | Team (rotating) | Standup |
| Sprint / milestone demo | End of each sprint | Delivery team + stakeholders | Project Manager / Eng Manager | Meeting |
| Stakeholder briefing | Monthly (or at milestones) | Executive sponsors + stakeholders | Project Manager | Email + deck |
| Pre-release readiness review | Per release | PM, QA, DevOps, Security, PdM | Release Manager | Meeting |
| Incident updates | As needed | PM, Eng Manager, Support, Sponsors | Project Manager / Release Manager | Slack + email |
| Retrospective | End of each sprint or milestone | Delivery team | Project Manager | Meeting |

## Communication Channels

| Channel | Purpose | Owner | Audience |
|---------|---------|-------|----------|
| Slack / team chat channel | Day-to-day collaboration and quick decisions | Engineering Manager | Delivery team |
| Email distribution list | Formal stakeholder updates and announcements | Project Manager | Stakeholders + sponsors |
| GitHub Issues / Project board | Work tracking and technical discussion | Developers / Project Manager | Engineering team |
| Release notes | Customer-facing and internal release communication | Technical Writer / Release Manager | All stakeholders + customers |
| Dashboard / reporting tool | Status metrics and project health | Data / Analytics Partner | PM, PdM, Sponsors |
| Design review tool (e.g., Figma) | Design feedback and specification | UX Designer | Developers, QA, PM |

## Escalation Path

| Situation | First Point of Contact | Escalation | Communications Owner |
|-----------|------------------------|------------|----------------------|
| Technical blocker | Engineering Manager / Tech Lead | Project Manager | Project Manager |
| Scope change request | Product Manager | Project Manager + Sponsors | Project Manager |
| Release risk or no-go | Release Manager | Engineering Manager + PdM | Release Manager |
| Security incident | Security / AppSec Representative | Engineering Manager | Project Manager |
| Customer escalation | Support / CS Representative | Product Manager | Project Manager |
